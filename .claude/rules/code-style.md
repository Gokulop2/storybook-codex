# Code Style Rules

## File naming
- All files must be kebab-case: `date-picker.tsx`, `user-profile.ts`, `api-client.ts`
- Applies to components, hooks, styles, tests, and config files
- **Exception — auto-generated icon files**: `packages/icons/src/components/` files are PascalCase (e.g. `Activity.tsx`, `AlarmClock.tsx`) because they are generated by SVGR from kebab-case SVG sources. The PascalCase matches the React component name and is required for JSX. Do NOT flag these as violations.

## Imports
- Alias React Aria Components primitives with `Aria*` when they collide with local names:
  ```ts
  import { Button as AriaButton, TextField as AriaTextField } from "react-aria-components";
  ```
- Icons: named imports only from `@opus2-platform/icons` (tree-shakeable)
- Internal components: import from `@/components`
- Utilities: import from `@/utils`

## Styling
- Tailwind CSS v4.1 only — no inline styles, no raw CSS outside theme files
- Use `sortCx()` for organized style objects, `cx()` for class merging
- `cx()` is a `tailwind-merge` wrapper (`extendTailwindMerge`) — it resolves Tailwind class conflicts correctly, not just concatenation
- Semantic color tokens only — never raw palette values:
  - ✅ `text-primary`, `fg-secondary`, `bg-brand-solid`, `border-primary`
  - ❌ `text-gray-900`, `bg-purple-600`
- Default transitions: `transition duration-100 ease-linear`
- Size variants: `sm`, `md`, `lg`, `xl`

## Custom typography scale
These are valid custom text size classes — do NOT flag them as unknown Tailwind classes:
- `text-display-xs`, `text-display-sm`, `text-display-md`, `text-display-lg`, `text-display-xl`, `text-display-2xl`

They are registered in `cx.ts` via `extendTailwindMerge` and defined as design tokens in `packages/ui/src/styles/theme.css`.

## Comments
- No comments unless the WHY is non-obvious
- No docstrings or multi-line comment blocks
- No "what the code does" comments — use well-named identifiers instead

## General
- No unused variables, imports, or dead code
- No error handling for scenarios that can't happen
- No features or abstractions beyond what the task requires
